# Инструкция для сотрудника контрагента (подрядчика)

Портал BillHub предназначен для подачи заявок на оплату, загрузки сопроводительных документов и отслеживания статуса согласования.

---

## 1. Вход в систему

1. Откройте портал по ссылке, предоставленной вашей организацией.
2. На странице входа введите **email** и **пароль**, выданные администратором.
3. Нажмите кнопку **Войти**.

После входа вы попадаете на главную страницу — **Заявки на оплату**.

---

## 2. Личный кабинет

Для перехода в личный кабинет нажмите на аватар в правом верхнем углу и выберите **Личный кабинет**.

В личном кабинете отображается:
- Ваш email
- Роль в системе

**Смена пароля:**
1. Введите текущий пароль.
2. Введите новый пароль (минимум 6 символов).
3. Повторите новый пароль.
4. Нажмите **Сохранить**.

Для выхода из системы нажмите на аватар и выберите **Выход**.

---

## 3. Заявки на оплату — обзор

Главная страница содержит таблицу всех ваших заявок с разделением по вкладкам:

| Вкладка | Содержимое |
|---------|-----------|
| **Все** | Все заявки вашей организации |
| **На рассмотрении** | Заявки, находящиеся на текущем этапе согласования |
| **Согласовано** | Одобренные заявки |
| **Отклонено** | Отклоненные заявки (доступна повторная отправка) |

### Колонки таблицы

- **Номер** — порядковый номер заявки
- **Объект** — объект строительства
- **Поставщик** — наименование поставщика (если указан)
- **Статус** — текущий статус заявки (цветной тег)
- **Оплата** — статус оплаты (РП / Оплачено)
- **Сумма** — сумма счета и сумма оплаченного
- **Дата создания** — дата подачи заявки
- **Срок** — количество дней с момента создания до согласования
- **Согласование** — прогресс-бар с этапами (Штаб, ОМТС)

### Фильтрация

Над таблицей доступны фильтры:
- **По объекту строительства** — выбор из списка
- **По статусу** — выбор из списка (на вкладке "Все")

---

## 4. Создание заявки

1. Нажмите кнопку **+ Добавить** в правом верхнем углу страницы.
2. Заполните форму:

| Поле | Обязательность | Описание |
|------|---------------|----------|
| Объект строительства | Обязательно | Выберите из списка активных объектов |
| Поставщик | Необязательно | Выберите из справочника |
| Срок поставки | Обязательно | Укажите количество дней и тип (рабочие или календарные) |
| Условия отгрузки | Обязательно | Выберите из выпадающего списка |
| Сумма счета | Обязательно | Введите сумму в рублях (должна быть больше 0) |
| Комментарий | Необязательно | Дополнительная информация по заявке |

3. **Прикрепите файлы** (минимум 1 файл обязателен):
   - Нажмите кнопку добавления файла.
   - Для каждого файла укажите **тип документа** из выпадающего списка.
   - Допустимые форматы: doc, docx, xls, xlsx, jpg, jpeg, png, tiff, tif, bmp, pdf.
   - Максимальный размер одного файла: 100 МБ.

4. Нажмите кнопку **Создать**.

После создания заявки файлы загружаются в фоновом режиме. Вы увидите сообщение "Заявка создана, файлы загружаются...". Статус загрузки отображается в таблице заявок.

**Контрагент** подставляется автоматически на основании вашей учетной записи — выбирать его вручную не требуется.

---

## 5. Просмотр заявки

Нажмите на строку заявки в таблице или на кнопку просмотра (иконка глаза) для открытия карточки заявки.

### Общая информация

В верхней части карточки отображаются:
- Номер и статус заявки
- Объект строительства и поставщик
- Срок поставки и условия отгрузки
- Сумма счета
- Комментарий (если указан)
- Дата создания

### Вкладка "Файлы"

Таблица загруженных файлов с колонками:
- Порядковый номер
- Имя файла
- Размер (для PDF — количество страниц)
- Тип документа

Доступные действия:
- **Просмотр** (иконка глаза) — предпросмотр файла (для PDF и изображений)
- **Скачать** (иконка стрелки) — скачать один файл
- **Скачать все файлы** — кнопка внизу таблицы, скачивает все файлы одним ZIP-архивом

Если заявка была переотправлена, в таблице появляется дополнительная колонка с информацией о том, кем и когда загружен файл.

### Вкладка "Согласование"

Хронологический лог событий по заявке:
- Дата отправки на согласование
- Решения по каждому этапу (Штаб, ОМТС): согласовано или отклонено
- При отклонении — причина (комментарий согласующего) и прикрепленные файлы с замечаниями
- Итоговый статус: согласовано / отклонено / отозвано

### Вкладка "Комментарии"

Чат для обсуждения заявки между вами и согласующими сотрудниками:
- Введите текст комментария и нажмите **Отправить** (или клавишу Enter).
- Вы можете **отредактировать** или **удалить** свой последний комментарий.
- Все участники видят автора и время каждого сообщения.

### Вкладка "Оплаты"

Информация о платежах по заявке (после согласования):
- Номер платежа, дата, сумма
- Статус: исполнена или планируется
- Прикрепленные файлы оплаты (с возможностью скачивания)

---

## 6. Отзыв заявки

Вы можете отозвать заявку, если она еще не была согласована.

1. В таблице заявок нажмите кнопку **Отозвать** (или откройте заявку и выберите действие).
2. При необходимости введите комментарий с причиной отзыва.
3. Подтвердите действие.

После отзыва заявка переходит в статус "Выведено" и больше не участвует в процессе согласования.

---

## 7. Повторная отправка после отклонения

Если заявка была отклонена на одном из этапов, вы можете доработать и отправить ее повторно.

1. Перейдите на вкладку **Отклонено**.
2. Нажмите кнопку **Отправить повторно** на нужной заявке.
3. В открывшемся окне вы можете изменить:
   - Срок поставки
   - Условия отгрузки
   - Сумму счета
4. При необходимости прикрепите **новые файлы** (для каждого укажите тип документа).
5. Напишите **комментарий** — укажите, что изменилось по сравнению с предыдущей версией.
6. Нажмите **Отправить**.

Заявка вернется на согласование с начала цепочки (Штаб -> ОМТС). Счетчик переотправок увеличится на 1.

---

## 8. Отслеживание статуса согласования

В таблице заявок колонка **Согласование** показывает прогресс в виде прогресс-бара:

| Состояние | Отображение |
|-----------|------------|
| На согласовании (Штаб) | Оранжевый прогресс, первый этап |
| На согласовании (ОМТС) | Оранжевый прогресс, второй этап |
| Согласовано (оба этапа) | Зеленый, 100% |
| Отклонено на этапе Штаб | Красный, 50% |
| Отклонено на этапе ОМТС | Красный, 100% |

Двухэтапное согласование:
1. **Штаб** — первичная проверка документов
2. **ОМТС** — финальное согласование

Если на каком-либо этапе заявка отклонена, вы увидите причину отклонения в карточке заявки на вкладке "Согласование" и сможете переотправить заявку с исправлениями.

---

## 9. Ошибки загрузки файлов

Если при загрузке файлов произошла ошибка, в таблице заявок рядом с заявкой отобразится красная иконка. Файлы загружаются в фоновом режиме, и процесс может занять некоторое время. При наличии ошибок попробуйте обновить страницу или обратитесь к администратору.

---

## 10. Краткая сводка доступных действий

| Действие | Доступно |
|----------|---------|
| Создать заявку | Да |
| Просмотреть заявку | Да |
| Отозвать заявку | Да (до согласования) |
| Переотправить отклоненную заявку | Да |
| Скачать файлы | Да (по одному и ZIP-архивом) |
| Написать комментарий | Да |
| Редактировать свой комментарий | Да (только последний) |
| Просмотреть историю согласования | Да |
| Просмотреть информацию об оплатах | Да |
| Редактировать заявку после создания | Нет (только через переотправку) |
| Удалить заявку | Нет (только отозвать) |
| Просмотреть справочники | Нет |
| Согласовать/отклонить заявку | Нет |
